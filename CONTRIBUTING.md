# Contributing to datasharing

Thank you for considering a contribution! This guide is a living document and community
improvements are welcome.

## Ways to contribute

- **Fix typos or grammar** – open a pull request directly.
- **Improve clarity** – if a section is confusing, suggest clearer wording.
- **Add examples** – real-world examples from different disciplines are especially
  valuable.
- **Update links** – external URLs go stale; please flag or fix broken ones.
- **Extend the R template** – improvements to `tidy_data_template.R` or
  `codebook_template.md` are welcome.

## Contribution workflow

1. [Fork](https://docs.github.com/en/get-started/quickstart/fork-a-repo) the repository.
2. Create a descriptive branch: `git checkout -b fix/broken-link-tidy-data`.
3. Make your changes.
4. Open a pull request against the `master` branch with a brief description of what you
   changed and why.

## Style guide

- Write in clear, jargon-free English accessible to non-statisticians.
- Use [GitHub-flavoured Markdown](https://docs.github.com/en/get-started/writing-on-github).
- Keep sentences short and prefer active voice.
- R code should follow the [tidyverse style guide](https://style.tidyverse.org/).

## Conduct

Please be respectful and constructive in all interactions. This project follows the
[Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
