# Contributing

Contributions to this guide are welcome! Here's how you can help:

## Ways to Contribute

- **Fix typos or grammar** – Small corrections are always appreciated.
- **Clarify existing content** – If a section is unclear, submit a pull request with improved wording.
- **Add examples** – Real-world examples make the guide more useful.
- **Translate** – Translations into other languages help a global audience.

## How to Submit Changes

1. Fork the repository.
2. Create a new branch: `git checkout -b my-improvement`
3. Make your changes to `README.md` or other files.
4. Commit your changes: `git commit -m "Brief description of change"`
5. Push to your fork: `git push origin my-improvement`
6. Open a pull request describing what you changed and why.

## Guidelines

- Keep the tone clear and accessible to a non-statistician audience.
- Cite sources when adding new factual claims.
- Be respectful and constructive in all discussions.

## Reporting Issues

If you find a factual error, broken link, or unclear section, please [open an issue](../../issues) with a description of the problem.
